뷰의 중앙저장소 -> Pinia(피냐)
각 컴포넌트의 공유 상태를 추출하여, 전역에서 참조할 수 있는 저장소에서 관리
Vue의 공식 상태 관리 라이브러리 -> 피냐 ~ ! 

Vite(비트) -> 프론트앤드 툴

stores 폴더 -> 중앙저장소 

피냐가 가져가는 부분 상태와 기능 

피냐(Pinia)의 구성요소 5가지 
'store' : 중앙저장소
'state' : 반응형 상태(데이터), ref() === state
'getters' :  계산된 값, computed() === getters
'actions' : 메서드, function() === actions
'plugin' : 추가 기능을 제공하거나 확장하는 도구나 모듈, 패키지 매니저로 설치 이후 별도 설정을 통해 추가 됨

Setup Stores의 반환 값 
pinia의 상태들을 사용하려면 반드시 반환해야 함 
중앙저장소의 목적은 모두에게 공유할 데이터를 모아주는 것이니깐 store에서는 공유 하지 않는 private한 상태 속성을 가지지 않음(목적에 맞는 데이터만 중앙저장소에 둬라)

actions은 getters와 달리 state 조작, 비동기, API 호출이나 다른 로직을 진행 가능 
반드시 액션을 사용하여 조작해야 함
