### 선형큐
- 1차원 배열을 이용한 큐
    - 큐의 크기 = 배열의 크기
    - front : 저장된 첫 번째 원소의 인덱스, rear : 저장된 마지막 원소의 인덱스 
- 상태 표현
    - 초기 상태 : front = rear = -1
    - 공백 상태 : front == rear
    - 포화상태 : rear = n - 1 

### 큐 구현
- 빈 리스트 
  - append, pop(0) 활용
  
- 크기가 정해진 리스트 
  - front, rear 사용
  
### 원형큐
- mod 나머지 연산자 사용 

### 연결리스트 
- 단순 연결 리스트(주소가 뒤에만)
  - 단점 : 역방향 탐색이 안됨, O(n)
  - 장점 : 메모리 동적 관리 가능, 삽입(처음, 끝에서는) O(1)이다.
- 이중 연결 리스트(주소가 앞 뒤)
  - 역방향 탐색 개선 
  
### 참고) deque(덱큐) 
- 장점 : popleft(), pop(0)처럼 사용 가능하고 시간 복잡도 : O(1) 
- 단점 : 슬라이싱 안됨, 정렬이 어려움, index 접근이 어려움
